# Calibration research: protocols and literature survey

Brief for chainlink #1 (adaptive ladder calibration) and #8 (personal me-space).
Status: protocol synthesis, not yet implementation.

The TL;DR up top: the user's instinct to frame #1 as "constrained optimization +
soft penalties + weighted regularizers" maps cleanly onto an existing genre
(Colorgorical, Palettailor, Tsitsulin's `coloropt`), and the trial-design
question has good answers in the suprathreshold-MLDS / Thurstonian-2AFC
literature. The interesting novelty for ansil16 is the dual-row taxonomy with
group-classifiability as an explicit constraint — not in any tool I found — and
the "tune an existing palette" framing (everyone else generates from scratch).

For #8, plenty of building blocks exist (QUEST/Ψ, MLDS, MacAdam-style ellipse
fitting, log-Euclidean SPD interpolation, CIECAM16) but no one has put them
together for personal ΔE on consumer displays. The 2022 Bujack PNAS result that
perceptual color space is non-Riemannian is a real warning shot for any
"warp space into unit spheres" plan and is worth designing around explicitly.

Top two things to pay attention to first:

1. **Bujack et al. 2022 (PNAS)** show diminishing returns violate Riemannian
   additivity for large color differences ([paper](https://www.pnas.org/doi/10.1073/pnas.2119753119)).
   That undermines #8's "warp ΔE into Euclidean" framing if pushed to
   suprathreshold scales. Solution: scope the metric to a small-difference
   regime around each anchor (where it *is* approximately Riemannian) and
   composite differently for big jumps.

2. **Tsitsulin's `coloropt`** ([blog](http://tsitsul.in/blog/coloropt/)) is the
   most direct prior art — multiple bounded palette "variants" (Normal, Bright,
   Dark, Fancy, Tarnish) generated by Powell's method on a composite loss with
   CIEDE2000 distances, grayscale, CVD, and bg-contrast terms. It already does
   most of what ansil16 calibration would do *for the per-tier palette*. But it
   does NOT enforce group-level classifiability (our C3), does NOT take an
   existing palette as starting point and minimize hue drift (our C1), and does
   NOT use per-observer trials. That gap is roughly the ansil16-specific bit.

---

## 1. Loss-function framing check

The user's instinct — express the problem as a weighted sum of a fidelity term,
several soft constraints, and a few hard bounds, then run a search — is exactly
the standard framing in palette-optimization papers. Colorgorical
([Gramazio et al. 2017](https://pubmed.ncbi.nlm.nih.gov/27875168/)) defines its
candidate scoring as a user-weighted sum over four components (Perceptual
Distance via CIEDE2000, Name Difference, Name Uniqueness, Pair Preference),
with hard bounds on L* (default [25, 85]) and optional hue filters. Palettailor
([Lu et al. 2020](https://arxiv.org/abs/2009.02969)) similarly composes three
objectives (point distinctness, name difference, color discrimination) and
optimizes via simulated annealing. Tsitsulin's `coloropt`
([blog post](http://tsitsul.in/blog/coloropt/)) explicitly enumerates five
objective terms (within-palette distinctness, grayscale distinguishability, CVD
robustness, achromatic background contrast, no-color-dominates uniformity) and
uses Powell's method on a composite loss inside bounded LCH boxes per palette
variant.

So the framing is unequivocally standard and the right vocabulary is "objective
function" / "score" with "weighted components" and "constraint boxes." The
slight twist worth noting: most papers treat *generation* as the task (start
from random or seed, search candidate space). The ansil16 calibrator task is
*tune an existing palette under tight constraint boxes around current
positions*, which is closer to a refinement / coordinate-descent / hill-climb
problem than the global search those tools do. That's a smaller search and
admits cheaper procedures (Powell, bounded BFGS, even pure 2AFC-driven
projected gradient) but the loss-function shape is the same.

## 2. Track A — Palette optimization landscape

### Per-paper notes

**Colorgorical** ([Gramazio, Laidlaw, Schloss 2017, IEEE TVCG](https://pubmed.ncbi.nlm.nih.gov/27875168/);
[live tool](http://vrl.cs.brown.edu/color)).
- Problem: generate N categorical colors balancing discriminability and aesthetic preference.
- Loss: weighted sum of four scores comparing each candidate color to the partial palette.
  - **Perceptual Distance**: CIEDE2000 in CIE Lab between candidate and each
    existing palette color (aggregate: min or mean — they use min based on the
    R wrapper and tool behavior).
  - **Name Difference**: from
    [Heer & Stone 2012](https://idl.uw.edu/papers/color-naming-models), built
    over the XKCD color-name survey; metric is a cosine-style distance between
    name-probability vectors.
  - **Name Uniqueness**: saliency of the candidate (how reliably named) from the
    same probabilistic model.
  - **Pair Preference**: predicted aesthetic preference of color pairs, from
    [Schloss & Palmer 2011](https://palmerlab.berkeley.edu/pdf/Schloss&Palmer(2011)sup.pdf).
- Search: iterative semi-random sampling. Each round, draw a batch of candidate
  colors from the bounded CIELAB region, score each against the current
  partial palette, sample next color weighted by score (not strict argmax —
  some stochasticity for diversity). Default L* ∈ [25, 85].
- Constraints: L* range, optional hue ranges, optional fixed colors via
  `startPalette`.
- User study: two experiments. Exp 1 verified score components were
  separable on 3/5/8-color palettes. Exp 2 compared Colorgorical output to
  ColorBrewer / Microsoft / Tableau / Random on discriminability + preference
  tasks. *Neither experiment used per-user 2AFC adaptive trials to tune
  weights.* The tool is fully algorithmic at runtime.
- Reusable for us: scoring decomposition, name-distance term (good proxy for C1
  hue identity), L*-range hard bounds.
- Not handled: dual-row taxonomy, per-slot identity preservation under
  perturbation, per-user calibration, terminal text-rendering context.

**Heer & Stone 2012** ([Stanford IDL page](https://idl.uw.edu/papers/color-naming-models)).
- Probabilistic color-naming model trained on XKCD survey. Two derived metrics:
  saliency (how reliably one names a color) and name distance (cosine-style
  distance between name-prob distributions).
- Directly relevant to C1 (nominal hue identity preservation): if a slot is
  "the red slot," the candidate's name-prob distribution should overlap with
  the original's. So C1 can be expressed as a Heer-Stone name-distance penalty
  between candidate and *original* slot color.
- Used by Colorgorical and Palettailor as the "Name Difference" component.

**Brewer / ColorBrewer**
([Brewer 2003 paper, tool: colorbrewer2.org](https://www.researchgate.net/publication/233640443_ColorBrewerorg_An_Online_Tool_for_Selecting_Colour_Schemes_for_Maps)).
- Hand-designed palettes (qualitative / sequential / diverging) with print and
  CVD considerations. The relevant transferable concept for us is the
  qualitative-with-tiers idea — ColorBrewer's "qualitative" sets are
  flat-categorical, but several cartography conventions stack qualitative on
  sequential (e.g., bivariate choropleths). No formal loss; designed by
  iteration with experts.
- Useful for us mostly as a sanity reference for "what hue spacing humans
  consider 6-distinct-categories" (their 6-color qualitative sets cluster
  remarkably similarly to the standard luv-rainbow row).

**Glasbey et al. 2007**
([Wiley](https://onlinelibrary.wiley.com/doi/abs/10.1002/col.20327);
[Python port](https://github.com/lmcinnes/glasbey)).
- Problem: max-distinguishable N colors from CIELAB.
- Loss: sequential maximization of Euclidean distance to nearest already-chosen
  color, or simulated annealing over fixed N for the joint problem.
- Constraints: bg color exclusion.
- Pure single-objective, very minimal — interesting only as a baseline; doesn't
  handle naming, preference, contrast, or tiers.

**Wijffelaars et al. 2008**
([CGF / Wiley](https://onlinelibrary.wiley.com/doi/abs/10.1111/j.1467-8659.2008.01203.x)).
- Univariate lightness-ordered palettes parameterized by hue, lightness,
  saturation, contrast, hue range, in CIELUV. Curves through color space.
- Not directly applicable (univariate-ordered, not multi-tier categorical) but
  the intuitive-parameters framing is good UX inspiration for the calibrator:
  expose few user-controllable knobs and tune the rest.

**Tsitsulin `coloropt`** ([blog](http://tsitsul.in/blog/coloropt/)).
- Five-component composite loss inside bounded LCH boxes. Five palette
  *variants* (Normal, Bright, Dark, Fancy, Tarnish) with different per-channel
  bounds. Powell's method, initial seed is permutations across the hue
  spectrum.
- Bounds (worth quoting):
  - Normal: H ∈ [0,360], C ∈ [50,75], L ∈ [40,75]
  - Bright: H ∈ [0,360], C ∈ [50,75], L ∈ [55,90]
  - Dark:   H ∈ [0,360], C ∈ [30,75], L ∈ [8,30]
  - Fancy:  H ∈ [0,360], C ∈ [15,40], L ∈ [40,75]
  - Tarnish: H ∈ [0,360], C ∈ [0,15],  L ∈ [30,70]
- The bounded-box-per-tier structure is **strikingly close** to ansil16's
  standard/emphasis dual rows. Notable: their Normal L ∈ [40,75] vs Bright
  L ∈ [55,90] uses L*-band separation for tier classifiability, exactly what
  the user wants to *abandon* — opens the question whether a radial coordinate
  gives an equivalent C3 signal without the rigidity. See section 5.
- No category-level classifiability term, no name-identity preservation, no
  per-user trials.

**Palettailor** ([Lu et al. 2020](https://arxiv.org/abs/2009.02969)).
- Joint creation + assignment for chart-data-aware colorization. Composite
  loss with point distinctness + name difference + color discrimination,
  simulated annealing. Validated by user study on scatterplots/lines.
- Not directly applicable (data-driven), but the SA-with-composite-loss
  pattern is.

**Lighter touch**: Bujack et al. 2022 PNAS
([article](https://www.pnas.org/doi/10.1073/pnas.2119753119)) demonstrates the
diminishing-returns / non-Riemannian property of perceptual color space —
implication for us: any sum-of-deltas approach systematically overestimates
large color differences. CIEDE2000 is fine *within* a constraint box (small
deltas), but objective terms that sum distances across many slot-pairs (e.g.
"mean discriminability") will exaggerate gains. Prefer min or soft-min
aggregations.

### Synthesis: a standard framework

Across the literature, the recipe is consistent:

```
L(palette) = − w_disc · DiscScore(palette)
           + w_name · NameTerm(palette)
           + w_pref · PairPref(palette)
           + w_contrast · BgContrastTerm(palette)
           + Σ I[bounds violated] · ∞
```

with components instantiated as:
- **Disc**: min over pairs of CIEDE2000 (or CAM02-UCS Euclidean, or ΔE_OK in
  Oklab) — min, not mean, because of Bujack.
- **Name**: Heer-Stone name distance for *separability* (between slots) and/or
  *identity* (vs. seed) for C1.
- **Contrast**: WCAG luminance ratio (or APCA Lc for self-luminous displays;
  see [APCA overview](https://www.accessibilitychecker.org/blog/apca-advanced-perceptual-contrast-algorithm/))
  between each palette slot and bg/fg.
- **Hard bounds**: per-channel boxes in LCH / CIELAB / Oklab.

Variations seen:
- Search: SA (Palettailor), Powell (coloropt), iterative weighted-sample
  (Colorgorical), simple greedy max-min (Glasbey).
- Color space: CIELAB + CIEDE2000 (Colorgorical, Glasbey, Palettailor); LCH
  derived from CAM02-UCS (coloropt); CIELUV (Wijffelaars); Oklab (newer work).
- Tiered structure: only coloropt does this, and only by L*-band separation.

## 3. Track B — Personal metric & psychophysics landscape

**MacAdam 1942**
([Wikipedia summary](https://en.wikipedia.org/wiki/MacAdam_ellipse)).
- 25 chromaticity points; observer set color-match variability defined an
  ellipse per point. Foundational for "perceptual space is not Euclidean in
  XYZ." Directly the template for #8's per-anchor measurement, with two
  differences: MacAdam used colorimeter matches (not 2AFC), and his 25 points
  are at fixed luminance.

**MLDS — Maloney & Yang 2003**
([JOV](https://jov.arvojournals.org/article.aspx?articleid=2192635);
[R package vignette](https://cran.r-project.org/web/packages/MLDS/vignettes/MLDS.pdf)).
- The right trial structure for suprathreshold perceptual scales. Observer
  sees two pairs (a,b) and (c,d) and judges which has the larger difference;
  the model fits a perceptual scale ψ such that
  P(decision = (c,d)) depends on |ψ(c)−ψ(d)| − |ψ(a)−ψ(b)| via a
  probit/logistic link. Triad variant: observer sees three stimuli, picks
  most-different.
- Very efficient: 100–200 trials gives a reliable scale along a 1D axis. For
  full ellipse fitting (4–6 directions × N anchors) this scales to thousands
  of trials, but each trial is cheap.
- Recent color-specific application: [Increasing magnitude of color differences amplifies category effects, 2024](https://pmc.ncbi.nlm.nih.gov/articles/PMC11176193/)
  uses MLDS triads directly for color difference perception. Validates the
  pattern.

**QUEST — Watson & Pelli 1983**
([Springer](https://link.springer.com/article/10.3758/BF03202828)) and
**Ψ — Kontsevich & Tyler 1999**
([PDF](https://www.psychophysics.ethz.ch/Downloads/protected/Kontsevich.pdf)).
- Bayesian adaptive staircases that converge on threshold (QUEST) or threshold
  + slope (Ψ) of a parametric psychometric function in ~30 trials for typical
  2AFC tasks. Standard machinery.
- Use case for us: in #8, threshold radius at each anchor in each direction
  (~30 trials × 6 directions × 30 anchors ≈ 5400 trials — too many for the
  ladder calibrator #1; appropriate budget for #8's deeper protocol).
- JavaScript implementation exists:
  [jsQuestPlus](https://link.springer.com/article/10.3758/s13428-022-01948-8),
  [StaircaseJS](https://github.com/hadrienj/StaircaseJS),
  [jspsych-staircase](https://github.com/kogpsy/jspsych-staircase).

**Levitt 1971**
([summary](https://www.semanticscholar.org/paper/Transformed-up-down-methods-in-psychoacoustics.-Levitt/e02c252f166e301b9bfb898c25220e82451d0442)).
- Non-Bayesian transformed up-down. 2-down-1-up converges to 70.7% correct,
  3-down-1-up to 79.4%, etc. Much simpler than QUEST, basically just rule-based
  on response history. For #1's ladder (where we want suprathreshold
  perceived-distance tuning, not threshold), this is over-applicable but
  worth knowing as a fallback.

**Thurstone 1927**
([reprint](https://parsmodir.com/wp-content/uploads/2013/02/thurstone1927.pdf)).
- Law of comparative judgment: paired comparisons decompose to scale values on
  a latent continuum, assuming normal-distributed perceived-value noise.
  Bradley-Terry-Luce is the logistic version. Formal underpinning of all
  2AFC scaling.

**Kruskal/Shepard MDS**
([Kruskal classic summary](http://cda.psych.uiuc.edu/psychometrika_citation_classic_summaries/kruskal_citation_classic_nonmetric_scaling.pdf)).
- Classic MDS recovers spatial configuration from pairwise dissimilarity
  judgments or odd-one-out triads. Less efficient than MLDS for a specific
  scale-fitting question but the canonical method for "what's the geometry
  here?" — relevant to #8 if we want a model-agnostic check on the warped
  space.

**CIECAM02 / CIECAM16** ([CIE 248:2022](https://cie.co.at/publications/cie-2016-colour-appearance-model-colour-management-systems-ciecam16);
[Wikipedia](https://en.wikipedia.org/wiki/CIECAM02)).
- Color appearance models predicting J/C/h (lightness/chroma/hue) under
  specified viewing conditions (illuminant, surround, adapting luminance).
- Question: does #8 partially reinvent CIECAM's adaptation handling? Answer:
  partially. CIECAM gives a population-average appearance prediction across
  viewing conditions; #8 builds an observer-and-condition-specific perceptual
  metric. CIECAM is the right *input space* for #8 (compute (a,b) anchor
  coordinates in CAM16-UCS rather than CIELAB) — it removes population-average
  adaptation variance — but the observer-specific residual is exactly what's
  left for #8 to measure. Use CIECAM16 / CAM16-UCS as the substrate, fit
  per-anchor ellipsoids on top.

**Asano & Fairchild 2016 individual colorimetric observer model**
([PLOS](https://journals.plos.org/plosone/article?id=10.1371%2Fjournal.pone.0145671)).
- Statistical model of individual CMFs based on Monte-Carlo over physiological
  parameters (cone peak wavelengths, optical-density variation, lens density,
  macular pigment). From display-based stimuli you *can* infer some of this
  (cone-fundamental tilt distinguishable from macular density) but cleanly
  separating contributions requires controlled monochromatic stimuli — not
  feasible on consumer displays. Implication for #8: don't try to fit
  observer CMFs; fit perceptual ellipsoids in display coordinates instead.

**Pant & Farup, Riemannian color metrics**
([Semantic Scholar](https://www.semanticscholar.org/paper/Riemannian-Formulation-of-the-CIEDE2000-Color-Pant-Farup/9d38b30fce810d3511b43c431b02f57d9e41aab3),
[HAL thesis](https://theses.hal.science/tel-00981484/document)).
- Treat ΔE formulas as line elements ds² = dx^T G(x) dx with metric tensor G
  varying smoothly over color space. Reformulates CIEDE2000 as Riemannian.
  Directly the math language for #8: each measured ellipsoid is a local SPD
  matrix G(anchor_i); we want a continuous G(x) interpolated over the anchor
  lattice.
- Chevallier & Farup 2018 add: naive (entrywise linear) interpolation of SPD
  matrices is geometrically wrong (doesn't preserve determinant / induces
  spurious anisotropy). Use Log-Euclidean interpolation (matrix log → linear
  interp → matrix exp) per
  [Arsigny et al. on log-Euclidean SPD metrics](https://www.researchgate.net/publication/6995152_Log-Euclidean_metrics_for_fast_and_simple_calculus_on_di_usion_tensors)
  or affine-invariant Riemannian interpolation — both well-established in
  diffusion-tensor MRI.

**Bujack et al. 2022 PNAS**
([article](https://www.pnas.org/doi/10.1073/pnas.2119753119)).
- Diminishing returns: big differences violate additivity, ruling out
  Riemannian geometry for perceptual color space at suprathreshold scale.
- The principled response for #8 is to scope the metric to a small-difference
  *local* regime (where Riemannian additivity is approximately valid) and
  either (a) accept that long geodesics in the warped space are heuristic, or
  (b) introduce a saturating link function so that big ΔE_me values plateau.

## 4. What's likely novel for ansil16

Things the literature does NOT cover that we'd build:

1. **Per-slot identity preservation under perturbation (C1).** Colorgorical's
   name-difference and `startPalette` come closest, but they enforce
   inclusion-of-fixed-colors, not "stay near a starting color within a soft
   penalty." Trivial to write but not off-the-shelf.

2. **Pairwise tier-ordering with respect to bg contrast (C2).** Every existing
   tool either ignores contrast-to-bg or uses it as a global term. Per-pair
   ordering ("for slot k, the emphasis version must have ≥ ε more contrast
   than the standard version") is unique to the dual-row design.

3. **Categorical group-classifiability as an explicit loss term (C3).** The
   closest analogue is coloropt's L*-band separation of Normal vs Bright, but
   that's a hard structural choice, not a measured perceptual constraint.
   ColorBrewer's qualitative+sequential composites are hand-designed. No tool
   *measures* group classifiability and optimizes for it.

4. **Tune-an-existing-palette-under-the-current-display framing.** All cited
   tools generate from scratch under standard-observer assumptions.
   Bring-your-own-palette + measure-the-actual-screen-under-actual-ambient is
   genuinely new for a terminal-palette context.

5. **In-context terminal-text-rendering trials.** The legibility-vs-
   discriminability distinction matters specifically because text is a thin-
   stroke, mid-spatial-frequency stimulus. Cited contrast literature
   ([APCA](https://www.accessibilitychecker.org/blog/apca-advanced-perceptual-contrast-algorithm/),
   [chromatic vs luminance CSF review](https://www.ncbi.nlm.nih.gov/pmc/articles/PMC7405764/),
   [Legge psychophysics-of-reading](https://legge.psych.umn.edu/sites/legge.psych.umn.edu/files/files/media/legge90_psychophysics_of_reading._xi._comparing_luminance_and_color_contrast.pdf))
   establishes the principle but not a calibration procedure that mixes
   threshold trials with rendered-text trials in a per-user loop.

6. **The me-space metric tensor on display-native coordinates.** Pant-Farup
   reformulates published ΔE formulas; we'd be fitting from raw human data on
   one display. Closer to MacAdam's original program than to anything since.

Things that are clearly NOT novel: 2AFC, MLDS, QUEST, weighted-sum composite
loss, simulated annealing or Powell on bounded boxes, CIEDE2000, name-distance
terms, CAM16-UCS as the substrate. Don't reinvent these.

## 5. Proposed protocol for #1 (the calibrator)

### Pre-conditions and scope

Inputs:
- Existing palette (the user's current `luv-rainbow-{dark,light}.conf`).
- The target display + ambient (user runs this where they normally work).
- Tier taxonomy: bg, fg, achromatic ramp {c0, c8, c7, c15}, standard row
  {c1..c6}, emphasis row {c9..c14}. Pairing: c_k ↔ c_{k+8} for k ∈ 1..6.

Outputs:
- Tweaked `.conf` with revised hex values. Slot positions, identities, and
  tier membership unchanged; just shifted within constraint boxes.

### Color-space substrate

Recommendation: **CAM16-UCS** (J', a', b' coordinates from CIECAM16 under the
display's actual white point + a coarse ambient estimate). Rationale:

- Better hue uniformity than CIELAB ([Wikipedia](https://en.wikipedia.org/wiki/CIECAM02)).
- Adaptation handled at the substrate level so we don't bake D65 assumptions
  into the loss when the user is in a tungsten room.
- ΔE_CAM16 ≈ Euclidean distance in CAM16-UCS, which lets us drop CIEDE2000's
  multiplicative-correction baggage.

**Oklab** ([Ottosson 2020](https://en.wikipedia.org/wiki/Oklab_color_space))
is a credible alternative — simpler math, available in CSS, slightly better
hue linearity than CIELAB — but lacks adaptation handling. For a tool meant to
calibrate against the actual viewing condition, CAM16 wins on principle. For
implementation ease, ship Oklab first and migrate.

### Polar palette parameterization

For each chromatic slot k ∈ {1..6}:
- Hue θ_k ∈ [0, 2π) — nominal hue identity (C1 anchor).
- Per-tier radial position r_k^standard, r_k^emphasis where "radial" means
  contrast-to-bg in CAM16-UCS (= |J' − J'_bg|, scaled into [0, 1]).
- Per-slot J' free within a band (no longer a per-row J' constraint).
- a', b' derived from (θ_k, chroma_k) once J' is fixed; chroma is what's left
  after C1 and C2 are satisfied.

For each achromatic slot ∈ {c0, c8, c7, c15}: just radial (contrast to bg),
hue undefined, chroma ≈ 0 enforced as hard bound (|a'|, |b'| < ε).

This **does** parameterize the design around (contrast-to-bg, hue) at the
user's request. It also accommodates light bg "emphasis is darker" naturally:
contrast is |ΔJ'|, so emphasis-row darker on light bg gives the same radial
direction as emphasis-row brighter on dark bg.

A note on whether the polar framing is really the best one: an alternative
is to keep (J', a', b') as free parameters and treat the polar coords as
*reporting* rather than *optimization* variables. The advantage is simpler
constraint boxes (axis-aligned in CAM16-UCS) and a familiar Cartesian gradient
surface. The disadvantage is that the user thinks polar, and constraint boxes
in CAM16-UCS don't naturally express "preserve hue identity ± 15°" without
trig. **Recommendation: optimize in (J', a', b') Cartesian under the hood, but
*express* C1/C2/C3 constraints in (contrast, hue, chroma) and check them after
each step.** That separates the loss-function design from the search geometry.

### Loss function

Let p be the candidate palette, p₀ the seed palette. Let J'(c), a'(c), b'(c) be
CAM16-UCS coords; let θ(c) = atan2(b', a'), r(c) = |J'(c) − J'(bg)|.

```
L(p) =   λ_id    · Σ_k [hue_drift(θ_k(p) − θ_k(p₀))]²            (C1)
       + λ_ord   · Σ_k max(0, ε_C2 − (r(c_{k+8}) − r(c_k)))      (C2)
       + λ_group · group_classifiability_penalty(p)              (C3)
       − λ_disc  · min_{i<j} ΔE_meas(c_i, c_j)                   (suprathreshold separability)
       + λ_legib · Σ_{(c, bg) pairs} max(0, contrast_min − |ΔJ'(c, bg)|)
       + λ_text  · in_context_legibility_penalty(p)              (optional, see below)
       + Σ box_violation_penalty(c_k)                            (hard bounds)
```

Term-by-term:

- **C1 hue identity, λ_id**:
  `hue_drift(Δθ) = (angle-wrap to [−π, π])(Δθ)`. Quadratic so small drift
  is cheap, large drift is expensive. Coefficient set so that 15° drift costs
  ~ same as one ΔE unit of discriminability lost.

- **C2 per-pair contrast ordering, λ_ord**:
  Hinge penalty on the radial-ordering invariant. ε_C2 is a small slack
  (say 5 CAM16-UCS units) below which the emphasis row "isn't visibly more
  contrasting." Asymmetric: zero penalty if ordering is satisfied with slack
  to spare.

- **C3 group classifiability, λ_group**:
  Two ways to operationalize:
  1. *Structural*: define a per-slot binary group label (0=standard,
     1=emphasis). Train a 1-NN classifier on (J', a', b') with leave-one-out
     CV. Penalty = 1 − LOO-accuracy.
  2. *Measured*: a small batch of "which group is this color from?" 2AFC
     trials interleaved into the session, with the classifier penalty
     calibrated against actual human accuracy.
  Start with (1); upgrade to (2) only if (1) is too coarse.

- **Discriminability, λ_disc**:
  Min-pairwise ΔE_meas. Use min (not mean) per Bujack to avoid summing-large-
  differences pathology. ΔE_meas is Euclidean in CAM16-UCS at first; the
  ladder-derived per-pair perceived-difference estimates (next section) refine
  this for pairs the user can actually compare.

- **Legibility (luminance-only), λ_legib**:
  Per [Legge](https://legge.psych.umn.edu/sites/legge.psych.umn.edu/files/files/media/legge90_psychophysics_of_reading._xi._comparing_luminance_and_color_contrast.pdf)
  and the [spatio-chromatic CSF literature](https://www.ncbi.nlm.nih.gov/pmc/articles/PMC7405764/),
  thin-stroke text needs luminance contrast; chromatic contrast at iso-
  luminance is ~2× weaker at all spatial frequencies. Hinge on |ΔJ'(c, bg)|
  against a calibrated minimum (start: 15 CAM16-UCS units, refine
  empirically). This is the *structural* answer to "should we measure
  legibility?": enforce a luminance-contrast floor that mathematically
  precludes the c4/c6 bug pattern from recurring.

- **In-context legibility, λ_text** (optional, see trial design below):
  Defer or skip. The structural luminance-contrast term handles the bg-vs-fg
  case; chromatic-on-chromatic legibility (c4 with c6 text) is best handled by
  forbidding iso-luminant pairs at all, not by measuring legibility per pair.

### Trial structure recommendation

The user's framing of the question is sharp: this is **suprathreshold scaling
under field conditions**, not threshold measurement. The trial structure should
match.

**Primary trial type: MLDS triads** ([Maloney & Yang 2003](https://jov.arvojournals.org/article.aspx?articleid=2192635)).

> Display three colors A, B, C as side-by-side patches. Ask "which is most
> different from the other two?"

Why this and not 2AFC pair-vs-pair:
- More information per trial: each triad implicitly compares three pairs.
- Direct fit to a perceptual scale via Maloney's MLE.
- Easy to render in HTML (three swatches).
- Already validated on color (cited
  [2024 study](https://pmc.ncbi.nlm.nih.gov/articles/PMC11176193/) used MLDS
  for color-difference perception).
- Symmetric so no ordering bias.

**Trial-mix recommendation** (~120 trials total, 8–12 min):

| Trial type            | Count | Purpose                                          |
| --------------------- | ----- | ------------------------------------------------ |
| MLDS triads (chromatic-only, within tier) | 50 | Tier-internal separation scale       |
| MLDS triads (cross-tier, hue-matched)     | 30 | Validate per-pair C2 ordering        |
| Group-classification 2AFC                 | 20 | Direct C3 measurement                |
| Text-on-bg legibility 2AFC                | 10 | Sanity check structural luminance floor |
| Hue-identity check (this color vs name)   | 10 | C1 validation                        |

Magnitude estimation (1–7 ratings) is rejected: scale-use bias is well-known,
and we don't need cardinal magnitudes — only ordinal comparisons.
Ranking/sorting tasks are powerful but slow and hard to render usably in
HTML; skip for v1.

In-context legibility trials: included as a small sanity batch (10 trials of
"read this short word") rather than as the main signal. Reasoning: it's
ecologically valid but high-variance (font-rendering, anti-aliasing, viewing
distance all matter), and the structural luminance-floor term in the loss
already enforces the actionable constraint. Use the in-context trials as a
post-hoc check: if all 10 are read correctly, the floor is calibrated; if not,
tighten it.

### Adaptive procedure

**Outer loop: bounded coordinate-descent / Powell** over the 17 chromatic
parameters (6 hues + 12 per-slot J's + 6 chromas, achromatic slots = 4 more
J's). Initial point: the existing palette. Step bounds: hue ±15°, J' ±10,
chroma ±10 in CAM16-UCS units. Powell's method per
[coloropt](http://tsitsul.in/blog/coloropt/) is appropriate here — derivative-
free, handles the discontinuous classifier penalty fine.

**Inner loop: refine ΔE_meas estimates from MLDS triad responses.**
MLDS gives a 1D scale ψ such that ΔE_meas(A, B) ≈ |ψ(A) − ψ(B)|. Trials
should be allocated by an active-learning rule that targets the slot pairs
currently driving the worst loss-component change. Rough sampler: each trial,
sample triads weighted by the gradient of the loss with respect to ΔE_meas at
that pair (= |λ_disc · indicator(this is the min pair)|, so just sample the
current min-pair plus 1-2 neighbors).

This is *not* a QUEST/Ψ threshold procedure because we're not finding a
threshold — we're estimating suprathreshold perceived distances. Don't import
QUEST machinery for the wrong task.

### Termination

- Stop when loss change over last 10 trials < 0.5% of initial loss.
- Hard cap: 150 trials or 15 minutes.
- Sanity floor: 60 trials before any termination check.

### Expected wall time

~10–12 minutes for an attentive user. Each trial: ~3s render + ~3s judgment
+ ~1s feedback/transition = 7s × 100 trials ≈ 12 min.

### Output format

A standard `.conf` file in the existing palette format, plus a sidecar
`<name>.calibration.json`:

```json
{
  "seed_palette": "luv-rainbow-light",
  "viewing_condition": {
    "white_point_xy": [],
    "adapting_lum_cd_m2": 0,
    "surround": "average"
  },
  "trial_log": [],
  "fitted_psi": [],
  "final_loss_components": {
    "id": 0, "ord": 0, "group": 0, "disc": 0, "legib": 0
  },
  "constraint_box_used": {}
}
```

Keep the sidecar so future #8 work can leverage the trials.

### Implementation skeleton for `designer/calibrate.html`

- Vanilla JS, no build, follows `rg-colors` pattern (per `CLAUDE.md`).
- Parse `.conf` from URL drop or file input.
- Color math: implement CAM16-UCS conversion in JS (~150 lines), or use
  [culori](https://culorijs.org/) inlined.
- Trial renderer: three swatches centered, "press 1/2/3 for most different."
- MLDS fitter: simple coordinate-ascent MLE in JS, no scipy needed for the
  trial counts we're talking about.
- Powell optimizer: ~50 lines or pull from
  [numeric.js](https://github.com/sloisel/numeric).
- Export: emit a new `.conf` blob + download link, plus the sidecar JSON.

## 6. Sketch for #8 (personal me-space)

### Build on QUEST/Ψ or roll our own?

**Build on Ψ.** Specifically, [QUEST+](https://jov.arvojournals.org/article.aspx?articleid=2611972)
generalized for joint slope+threshold per-direction, or
[jsQuestPlus](https://link.springer.com/article/10.3758/s13428-022-01948-8)
straight off the shelf. Don't reinvent. We *do* want threshold estimates
(unlike #1) because the ellipsoid axes are defined by perceptual JNDs in each
direction.

### Anchor lattice and ellipsoid fitting

Per the user's brief: 30–50 anchors in (a', b') at 2–3 J' levels.
At each anchor: 4–6 directions, threshold radius via Ψ (~30 trials each), fit
a 3D ellipsoid (3×3 SPD metric tensor) by Gauss-Newton on the principal-axis
form. Output: G(anchor_i) for each anchor.

### Metric tensor interpolation

Use **log-Euclidean interpolation** ([Arsigny et al.](https://www.researchgate.net/publication/6995152_Log-Euclidean_metrics_for_fast_and_simple_calculus_on_di_usion_tensors))
or affine-invariant Riemannian. For each query color x:

1. Find k-nearest anchors in CAM16-UCS.
2. For each, compute matrix log of G(anchor_i).
3. Weighted average (by distance kernel) in log space.
4. Matrix-exp back to SPD.
5. ΔE_me(x, y) ≈ sqrt((y−x)^T G(midpoint(x,y)) (y−x)).

Pant & Farup's Riemannian-CIEDE2000 formulation
([paper](https://www.semanticscholar.org/paper/Riemannian-Formulation-of-the-CIEDE2000-Color-Pant-Farup/9d38b30fce810d3511b43c431b02f57d9e41aab3))
is the right model for the math, but our G(x) is *measured*, not derived from
a formula.

### Is CIECAM a partial substitute?

Yes, partially. CAM16-UCS provides the right substrate (handles adaptation,
gives perceptually-better hue uniformity than CIELAB) but bakes in population-
average chromatic-discrimination scaling. For an off-the-shelf top-decile user,
the residual ellipsoids over CAM16-UCS may be small enough that the marginal
benefit of measuring them is modest. **Suggest a pre-flight calibration: 50
quick MLDS triads scattered across the (a',b') plane to estimate global scale
correction. If the user is within ~1.2× of population-average CAM16-UCS scale
everywhere, declare CAM16-UCS adequate and skip the full #8 protocol. If
local anisotropy is large (say >2× elongation), full #8 is warranted.**

### Non-Riemannian caveat

[Bujack et al. 2022](https://www.pnas.org/doi/10.1073/pnas.2119753119): big
differences violate additivity. Mitigation: explicitly scope the metric to
"local" (within ~5 ΔE units of any anchor) and use a saturating composite
function (e.g., ΔE_global(x,y) = κ · tanh(integral / κ)) for global queries.
Document the limitation; for the ANSI palette use case, all 16 slots fit
within a regime where the linear approximation holds.

### Worth building?

For a top-decile user: maybe modestly. The bigger payoff is documenting *one*
display's per-anchor ellipsoid map and seeing whether the warping is small
(in which case CAM16-UCS suffices, modulo a uniform scale) or large (in which
case we have a publishable single-observer dataset). Suggest #8 as a research
project, not a hard prerequisite for #1.

## 7. Open questions

1. **Empirical**: is the L*-band coloropt-style C3 (separable by J' alone) so
   strongly preferred by users that the proposed (radial-coord) C3 measurably
   degrades classifiability? Test: render both versions, run 50 group-2AFC
   trials. Decide after data.

2. **Method**: should the MLDS triads be drawn uniformly over all
   `C(16, 3)` triads or biased to triads involving the current min-
   discriminability pair? Active-learning bias likely better but un-
   measured here; literature gap. Consider a hybrid: 50/50 mix.

3. **Loss-weight calibration**: I gave qualitative weights (15° hue drift ≈
   1 ΔE) but no principled procedure for picking λs. Standard answer is
   cross-validation against held-out trials, but our trial budget per session
   doesn't support a clean train/test split. Suggest: ship with author-chosen
   λs, expose them in the calibrator UI, let the user retune via "this
   palette looks worse" feedback.

4. **CAM16 viewing-condition parameters**: how does the calibrator know the
   adapting luminance and surround? Realistic answer: ask the user "is this
   a bright room? dim? dark?" → 3-way categorical → CAM16 surround
   parameter. Adapting luminance is harder; assume 60 cd/m² (typical office)
   and document the assumption. Or: skip CAM16, ship Oklab v1, upgrade later.

5. **Display gamut and calibration**: we're calibrating against the
   *displayed* color, but we don't have ICC profiles for terminal use. ΔE_meas
   thus rolls together display rendering errors and human perceptual errors.
   For a self-calibration tool that's fine (the user perceives what they
   perceive). For #8's broader claim ("personal me-space") this is a
   confound — needs a colorimeter for a clean separation. Document as a
   limitation.

6. **Group-classifiability via 1-NN**: arbitrary classifier choice. Could use
   LDA, logistic regression on (J', a', b'), or k-NN with larger k. Probably
   doesn't matter at our N (6 colors per group × 2 groups = 12 points) but
   worth checking robustness on synthetic data before deploying.

7. **In-context trials**: actual rendered text on macOS Terminal vs kitty vs
   iTerm2 will subtly differ (subpixel rendering, ClearType variants). For a
   per-machine calibration we'd want to render in the actual terminal — but
   the designer is HTML. Possible answer: a small Python/native helper that
   renders to a screenshot and embeds it in the page. Worth filing as
   chainlink issue if pursued.

8. **Does the user actually want both rows preserved at 6 slots?** A
   plausible alternative palette structure is 12 distinct hues at uniform
   contrast (one row) with bg/fg + grayscale ramp making up 4 more. Worth
   surfacing as a separate design conversation; not a calibrator question.
